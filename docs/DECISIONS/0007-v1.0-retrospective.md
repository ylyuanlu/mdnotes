mdnotes 项目经过 5 个迭代周期完成 v1.0.1 稳定版。本文档总结关键技术决策、流程教训、以及对未来阶段的建议。

## 关键技术决策

继承 ADR-0001~0006：

- **ADR-0001** Markdown 渲染 pipeline（html2text）
- **ADR-0002** SQLite WAL journal mode
- **ADR-0003** DB 路径 `~/.mdnotes.db`（user-level）
- **ADR-0004** count SQL 预埋 `deleted_at IS NULL`（v1.0 后续 soft-delete 接口）
- **ADR-0005** FTS5 vs LIKE（采纳 FTS5）
- **ADR-0006** search scope（standalone FTS5 + manual trigger，受 SQLite 3.53.1 约束）

### v1.0 新增决策点

- **SQLite 3.53.1 约束**：`content='notes'` 关联型 FTS5 auto-sync 不工作 → 降级为 standalone FTS5 + 3 manual trigger（`notes_ai` / `notes_au` / `notes_ad`）
- **BM25 排序权重**：`bm25(notes_fts, 10.0, 1.0, 1.0)` — title 权重 10x
- **WAL 模式 + mmap + cache 组合**：P95 从 183.6ms 降至 1.3ms（140x 性能提升）

## 流程教训（关键 5 条）

### 1. 派发最小信息铁律（2026-06-21）

派 sub-agent 只传任务意图，不传可选参数（model / cwd / sandbox / thinking / context:fork / 详细 PRD）。

**教训来源**：2026-06-21 凌晨派 dev retry 显式传 `model=M2.7` → 站到 fallback 链末端 → M2.7 抽风没得切 → 7 小时断链。

**未来执行**：所有派发一律走默认配置；sub-agent 自主从 task.json 读上下文。

### 2. dev 必 commit-on-branch（2026-06-20）

dev 写完代码**必须**自己 commit 到分支，main 不兜底代 commit。

**教训来源**：v1.0.1 P95 fix / UX fix / retry-coverage 3 次 working tree 残留，main 兜底代 commit 2 次 = 违反"agent 自治"原则。

**未来执行**：dev AGENTS.md 已有 §77/95/138 规定；reviewer / qa 验收前必须确认 `git log` 有 dev commit。

### 3. 禁止硬编码本机绝对路径（2026-06-21）

代码 / 测试 / 配置 / fixture 禁止出现 `/home/<user>/...` / `C:\...` / `/Users/<user>/...`。

**教训来源**：mdnotes v1.0.1 GHA workflow 跑 3 次才全过：lint ruff / hardcode 路径（CI runner 无主公本机路径）/ `uv` 未装（CI runner 裸 ubuntu）。

**未来执行**：dev / qa AGENTS.md 已加铁律 4 + grep 自查命令。

### 4. 升版铁律（2026-06-19）

主公没明确说"进入 v0.X 阶段"前，不出现 v0.X 字眼（X≠1）。v0.1 / v0.5 / v1.0 / v1.0.1 由主公明确指令触发；agent 自己永远不能升版。

**未来执行**：本帅以后不再主动提"v0.X 下一阶段"，v1.0 已发后下一阶段叫 v1.5。

### 5. 结构变更先方案（2026-06-21）

AGENTS.md / MEMORY.md / TOOLS.md / 范式设计文档改动前**必须**先发方案给主公拍板。

**未来执行**：本帅以后不再擅自改核心文档；改 agent workspace AGENTS.md 也要先报方案。
